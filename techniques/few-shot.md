# Few-shot Prompting

## Суть
Даёшь модели примеры формата — она понимает паттерн и повторяет его.
Модель учится не из описания, а из демонстрации.

## Когда применять
- Нужен строго определённый формат ответа
- Классификация по категориям
- Задачи где описать формат словами сложнее чем показать пример

## Когда не работает
- Задача слишком простая — примеры только перегружают
- Примеры плохого качества — модель учится плохому паттерну
- Пример слишком конкретный — модель копирует структуру вместо смысла
- Творческие задачи — примеры делают ответ шаблонным

## Формула
"[Пример 1] → [Результат 1]
[Пример 2] → [Результат 2]
[Новый входной текст] → [?]"

## Пример из практики

**Задача:** классификация отзывов покупателей.

**Без Few-shot:**
Модель сама решает как классифицировать — результат непредсказуем.

**С Few-shot:**
"Доставка быстрая, товар отличный" → Позитивный
"Сломалось на второй день" → Негативный
"Всё нормально, без восторгов" → [?]

Результат: Нейтральный — точно в нужном формате.

## Критический урок из практики
Few-shot с точным примером готового текста заставил модель
копировать структуру вместо переработки входного текста.

Решение: заменить готовый пример на список требований к стилю.
Результат вырос с 9/10 (копирование) до 9/10 (реальная переработка).

## Ключевой вывод
Few-shot показывает формат, не содержание.
Если пример слишком конкретный — модель копирует, а не думает.
Давай пример стиля, не пример структуры.
