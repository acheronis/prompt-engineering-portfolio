# Система промптов для поддержки клиентов

Промпт-система для интернет-магазина одежды.
Покрывает 5 наиболее частых типов обращений в поддержку.

## Проблема

Команда поддержки обрабатывала 50+ одинаковых обращений в день.
Среднее время ответа вручную: 3–5 минут на тикет.
Итого: 250+ минут ежедневно на повторяющуюся работу.

## Решение

Разработана система промптов с явными ограничениями под каждый тип обращения.
Каждый промпт устраняет типичные сбои: размытые формулировки,
лишние условия, неправильное направление действий.

## Результаты

| Тип обращения | До | После | Рост |
|---|---|---|---|
| Статус заказа | 6/10 | 9/10 | +3 |
| Возврат товара | 7/10 | 8/10 | +1 |
| Подбор размера | 4/10 | 9/10 | +5 |
| Оплата при получении | 7/10 | 9/10 | +2 |
| Скидки и акции | 8/10 | 10/10 | +2 |
| **Среднее** | **6.4/10** | **9.0/10** | **+2.6** |

**Общий рост качества: +40%**

## Метод оценки

Каждый ответ оценивался по 4 критериям:
точность, тон, структура, actionability.
Baseline тестировался в чистых сессиях без переноса контекста.

## Ключевые решения

**Ограничения вместо примеров.**
Few-shot примеры вызывали структурное копирование без адаптации к запросу.
Явные ограничения дали более гибкие и точные результаты.

**Инъекция доменного контекста.**
Без роли модель давала общие ответы на неоднозначные запросы.
Role + контекст полностью устранил эту проблему.
