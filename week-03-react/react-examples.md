# Неделя 3 — ReAct (Reason + Act)

## Что это
Модель думает → действует → проверяет результат → повторяет цикл.
Отличие от CoT: есть петля проверки, а не просто цепочка мыслей.

## Урок из практики
Claude блокирует формат "Мысль/Действие/Наблюдение".
Решение: использовать "Шаг 1 — Анализ / Шаг 2 — Проверка" — результат тот же.
Вывод: Prompt Engineer адаптирует технику под модель, а не копирует шаблон.

## Пример — План обучения навыку

**Промпт A (без техники):**
Я хочу выучить новый навык за 3 месяца, у меня есть 1 час в день. Как составить план?

**Промпт B (ReAct):**
Тот же вопрос + структура:
Шаг 1 — Анализ ситуации
Шаг 2 — Проверка ограничений
Шаг 3 — Составление плана
Шаг 4 — Проверка плана на слабые места
Финальный ответ

## Разница
Промпт A — сразу список советов, без проверки.
Промпт B — модель нашла слабые места плана и исправила их до финала.

## Вывод
ReAct нужен когда важно не просто ответить, а проверить ответ перед выдачей.
