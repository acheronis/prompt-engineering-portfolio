# Chain-of-Thought (CoT)

## Суть
Модель рассуждает шаг за шагом перед финальным ответом.
Без CoT модель прыгает к ответу и может ошибиться.
С CoT — показывает логику, ошибку видно сразу.

## Когда применять
- Сложные задачи с несколькими условиями
- Анализ ситуации с множеством факторов
- Задачи где важна логика вывода, а не только результат

## Когда не нужен
- Простые фактические вопросы
- Генерация контента без аналитики
- Задачи где нужен короткий прямой ответ

## Формула
Добавь в промпт:
"Думай шаг за шагом. Сначала [что анализировать], потом [что сделать]."

## Пример из практики

**Задача:** составить план обучения навыку за 3 месяца.

**Без CoT:**
Модель сразу выдаёт список советов без проверки логики.

**С CoT:**
Шаг 1 — Анализ ситуации: 1 час в день = 90 часов за 3 месяца.
Шаг 2 — Проверка ограничений: риск потери мотивации на 2-3 неделе.
Шаг 3 — Составление плана: месяц 1 база, месяц 2 практика, месяц 3 проекты.
Шаг 4 — Проверка плана: добавить мини-результаты каждую неделю.
Финальный ответ: структурированный план с учётом всех ограничений.

## Ключевой вывод
CoT даёт мало на простых темах.
На сложных задачах с рассуждением — критичен.
Чем сложнее задача, тем важнее CoT.
