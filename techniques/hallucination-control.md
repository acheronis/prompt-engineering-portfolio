# Контроль галлюцинаций

## Суть
Модели уверенно выдают неверные факты если их не остановить.
Правильный промпт заставляет модель признавать неопределённость
вместо того чтобы придумывать данные.

## Когда критично
- Финансовые данные и цифры
- Даты, имена, названия компаний
- Юридические и медицинские факты
- Любые данные которые будут использованы в production

## Техники контроля

### 1. Явное разрешение на незнание
Добавь в промпт:
"Если не уверен в точности — прямо скажи об этом.
Не придумывай факты которых не знаешь точно."

### 2. Верификация через второй промпт
Сначала получи ответ.
Потом запусти второй промпт:
"Проверь этот ответ на фактические ошибки и неточности: [ответ]"

### 3. Ограничение области знаний
"Отвечай только на основе предоставленных данных.
Если данных недостаточно — скажи что не знаешь."

### 4. Self-Consistency
Запусти один и тот же промпт 3 раза.
Если ответы расходятся — данные ненадёжны.

## Пример из практики

**Задача:** узнать инвесторов Anthropic с точными суммами.

**Без защиты (Промпт A):**
Модель уверенно выдала цифры без оговорок.
Данные по Google суммированы некорректно — нет предупреждения.

**С защитой (Промпт B):**
Добавлены инструкции: "если не уверен — скажи об этом".
Результат: модель добавила ⚠️ к данным по Google,
написала "данные могут быть неточными",
разделила суммы по годам для прозрачности.

## Сравнение

| | Без защиты | С защитой |
|---|---|---|
| Неточные данные | Выдаёт уверенно | Помечает ⚠️ |
| Отсутствующие данные | Придумывает | Говорит "не знаю" |
| Доверие к результату | Низкое | Высокое |
| Безопасность для бизнеса | Риск | Контролируемо |

## Ключевой вывод
Модель не знает что она не знает — если её не спросить.
Явное разрешение на незнание — самый простой и эффективный способ
снизить количество галлюцинаций без изменения архитектуры.
