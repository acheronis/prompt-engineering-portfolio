# ReAct (Reason + Act)

## Суть
Модель проходит цикл: анализ → действие → наблюдение → повторение.
Отличие от CoT: есть петля проверки, а не просто цепочка мыслей.
Модель не просто думает — она проверяет себя перед финальным ответом.

## Когда применять
- Задачи где важно не пропустить детали
- Многошаговые решения с несколькими требованиями
- Когда нужна проверка ответа до его выдачи

## Когда не нужен
- Простые однозначные задачи
- Генерация контента по шаблону
- Задачи где достаточно одного шага

## Формула
"Пройди по шагам:
Шаг 1 — Анализ: [что оцениваешь]
Шаг 2 — Проверка: [что может пойти не так]
Шаг 3 — Решение: [конкретные действия]
Шаг 4 — Проверка решения: [слабые места]
Финальный ответ: [итог]"

Важно: Claude блокирует формат Мысль/Действие/Наблюдение.
Используй Шаг 1/Шаг 2 — результат тот же.

## Пример из практики

**Задача:** ответить на негативный отзыв клиента.

**Без ReAct (6/10):**
Модель сразу пишет ответ. Просит клиента самого найти контакты.

**С ReAct (10/10):**
Шаг 1 — Понять проблему: цвет не тот + задержка 2 недели.
Шаг 2 — Оценить серьёзность: клиент может уйти к конкуренту.
Шаг 3 — Выбрать тон: официально-дружелюбный, берём ответственность.
Шаг 4 — Написать ответ: извинения + контакты компании + решение.
Результат: модель не упустила ни одной детали.

## Ключевой вывод
ReAct нужен когда важно не просто ответить, а проверить ответ перед выдачей.
Пошаговый разбор не позволяет пропустить детали.
