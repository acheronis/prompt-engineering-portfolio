# Structured Output — JSON извлечение из текста

Система промптов для извлечения структурированных данных из свободного текста.
Применение: автоматическая обработка обращений, классификация, маршрутизация тикетов.

## Проблема

Клиенты пишут в свободной форме.
Системы обработки требуют структурированные данные.
Ручная классификация — медленно и дорого.

## Решение

Промпты которые извлекают структурированный JSON из любого текста.
Модель парсит свободный текст и возвращает валидный JSON без лишнего текста.

## Пример 1 — Анализ негативного отзыва

**Промпт:**
Проанализируй отзыв клиента и верни результат строго в JSON формате.
Никакого текста до или после JSON — только валидный JSON.

Отзыв: [текст отзыва]

Верни JSON в формате:
{
  "sentiment": "positive/negative/neutral",
  "main_problem": "главная проблема одним предложением",
  "urgency": "high/medium/low",
  "recommended_action": "что должен сделать менеджер",
  "response_tone": "official/friendly/apologetic"
}

**Входные данные:**
"Заказал худи, ждал 2 недели, пришло не того цвета. Ужасный сервис."

**Результат:**
{
  "sentiment": "negative",
  "main_problem": "Клиент получил заказ с задержкой и неправильным цветом товара",
  "urgency": "high",
  "recommended_action": "Связаться с клиентом, извиниться, предложить замену товара или возврат средств и ускоренную доставку",
  "response_tone": "apologetic"
}

## Пример 2 — Классификация обращения в поддержку

**Промпт:**
Ты система анализа входящих обращений интернет-магазина.
Проанализируй обращение клиента и верни строго валидный JSON.
Никакого текста до или после — только JSON.

Обращение: [текст обращения]

Верни JSON в формате:
{
  "request_type": "return/exchange/complaint/question",
  "order_number": "номер заказа или null",
  "product": "товар или null",
  "reason": "причина обращения",
  "payment_method": "card/cash/unknown",
  "priority": "high/medium/low",
  "next_step": "конкретное следующее действие для менеджера",
  "auto_resolvable": true/false
}

**Входные данные:**
"Здравствуйте, хочу вернуть брюки, они мне малы. Заказ №4521. Оплачивал картой."

**Результат:**
{
  "request_type": "return",
  "order_number": "4521",
  "product": "брюки",
  "reason": "товар мал по размеру",
  "payment_method": "card",
  "priority": "medium",
  "next_step": "Отправить клиенту инструкцию по возврату и уточнить удобный способ передачи товара",
  "auto_resolvable": true
}

## Ключевые правила JSON промптов

**1. Явный запрет лишнего текста.**
"Никакого текста до или после JSON — только валидный JSON."
Без этого модель добавляет пояснения которые ломают парсинг.

**2. Точная схема с типами.**
Указывай допустимые значения: "positive/negative/neutral".
Без этого модель изобретает свои значения.

**3. Null для отсутствующих данных.**
"номер заказа или null" — модель не придумывает данные которых нет.

**4. Роль системы.**
"Ты система анализа..." — задаёт контекст обработки, не разговора.

## Применение

- Автоматическая маршрутизация тикетов поддержки
- Классификация и приоритизация обращений
- Извлечение данных из неструктурированного текста
- Интеграция с CRM и системами обработки заявок
