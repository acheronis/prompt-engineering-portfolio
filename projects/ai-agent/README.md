# AI Agent — Обработка жалоб интернет-магазина

Агентная система для автоматической обработки жалоб клиентов.
Агент самостоятельно анализирует ситуацию, выбирает инструмент
и принимает решение на основе правил бизнес-логики.

## Чем агент отличается от обычного промпта

Обычный промпт: вопрос → ответ.
AI агент: анализ → выбор инструмента → действие → следующий шаг.

Агент сам решает что делать дальше на основе результата предыдущего шага.

## Архитектура агента

Три инструмента:
- CHECK_ORDER — проверка статуса заказа
- ISSUE_REFUND — оформление возврата
- ESCALATE — передача старшему менеджеру

Правила принятия решений:
- Ошибка доставки → предложи возврат или замену
- Сумма возврата больше 5000₽ → эскалируй менеджеру
- Агрессивный клиент → эскалируй менеджеру
- Всегда проверяй заказ перед любым действием

## Финальный промпт

Ты AI агент по обработке жалоб интернет-магазина.
У тебя есть три инструмента:

ИНСТРУМЕНТ 1 — CHECK_ORDER(номер заказа)
Проверяет статус заказа в системе.

ИНСТРУМЕНТ 2 — ISSUE_REFUND(номер заказа, сумма)
Оформляет возврат средств.

ИНСТРУМЕНТ 3 — ESCALATE(номер заказа, причина)
Передаёт жалобу старшему менеджеру.

Правила принятия решений:
- Если заказ доставлен с ошибкой → предложи возврат или замену
- Если сумма возврата больше 5000₽ → эскалируй менеджеру
- Если клиент агрессивен → эскалируй менеджеру
- Всегда сначала проверяй заказ перед любым действием

Покажи рассуждения в формате:
МЫСЛЬ: [что анализируешь]
ДЕЙСТВИЕ: [какой инструмент и почему]
РЕЗУЛЬТАТ: [что получил]
СЛЕДУЮЩИЙ ШАГ: [что делаешь дальше]

## Тестовый кейс

Жалоба: "Заказ №8834. Заплатил 7500₽ за куртку, прислали не тот размер.
Хочу деньги обратно. Это безобразие!"

Результат агента:

МЫСЛЬ: Клиент жалуется на неправильный размер. Сумма 7500₽.
Сначала проверяю заказ.
ДЕЙСТВИЕ: CHECK_ORDER(8834)
РЕЗУЛЬТАТ: Заказ доставлен, ошибка подтверждена.

МЫСЛЬ: Сумма возврата 7500₽ превышает лимит 5000₽.
Автоматический возврат невозможен.
ДЕЙСТВИЕ: ESCALATE(8834, "не тот размер, сумма возврата 7500₽")
РЕЗУЛЬТАТ: Жалоба передана старшему менеджеру.

СЛЕДУЮЩИЙ ШАГ: Уведомить клиента о передаче жалобы.

## Анализ решений агента

Агент принял три правильных решения самостоятельно:

1. Проверил заказ перед действием — соблюдено обязательное правило
2. Определил что сумма 7500₽ больше лимита 5000₽
3. Выбрал ESCALATE а не ISSUE_REFUND — правильно по бизнес-логике

## Ключевые инженерные решения

**Явные правила принятия решений.**
Без правил агент импровизирует и делает ошибки.
Чёткие условия "если → то" дают предсказуемое поведение.

**Формат МЫСЛЬ/ДЕЙСТВИЕ/РЕЗУЛЬТАТ.**
Заставляет агента показывать рассуждения.
Ошибку в логике видно до того как агент совершил действие.

**Обязательный первый шаг.**
"Всегда сначала проверяй заказ" — исключает действия без проверки данных.
Критично для систем где действие необратимо (возврат денег).

## Применение

- Автоматическая обработка жалоб и возвратов
- Триаж обращений в поддержку
- Квалификация лидов с маршрутизацией
- Любой процесс с условной логикой и несколькими возможными действиями
