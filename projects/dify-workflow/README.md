# Dify Workflow — цепочка обработки обращений

Многошаговый AI workflow в Dify для классификации и обработки
клиентских обращений без участия человека.

## Архитектура

User Input → Question Classifier → (2 ветки)
Ветка 1: Knowledge Retrieval → LLM → Output
Ветка 2: Output (эскалация менеджеру)

## Компоненты

**User Input** — получает вопрос клиента
**Question Classifier** — определяет тип обращения
**Knowledge Retrieval** — ищет ответ в базе знаний
**LLM** — формирует финальный ответ по контексту
**Output** — возвращает ответ или эскалирует

## Системный промпт LLM

Ты ассистент поддержки интернет-магазина одежды.
Отвечай ТОЛЬКО на основе базы знаний.
Если ответа нет — скажи: "Эта информация недоступна.
Обратитесь к менеджеру."

## Классы Question Classifier

Класс 1: доставка, возврат, размеры, оплата
Класс 2: всё остальное → эскалация

## Ключевые решения

**Question Classifier вместо IF/ELSE.**
Классификатор понимает смысл вопроса а не ищет ключевые слова.
Более гибкий и точный для реальных обращений.

**Knowledge Retrieval как контекст LLM.**
Найденные фрагменты из базы знаний передаются в LLM как контекст.
Модель отвечает только по реальным данным.

## Применение

- Автоматическая обработка обращений 24/7
- Умная маршрутизация по типу запроса
- Масштабируемая система без дополнительных ресурсов
