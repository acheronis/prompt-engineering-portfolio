# Неделя 6 — Prompt Comparison Framework

## Задача
Написать ответ на негативный отзыв клиента:
"Заказал худи, ждал 2 недели, пришло не того цвета. Ужасный сервис."

## Результаты сравнения

| Техника | Оценка | Проблема |
|---|---|---|
| 1 — Без промпта | 6/10 | Просит клиента искать контакты сам |
| 2 — Role + Specificity | 5/10 | Слишком коротко, та же проблема |
| 3 — Few-shot | 9/10 | Пример исправил проблему с контактами |
| 4 — CoT | 9/10 | Анализ причин сделал ответ живым |
| 5 — ReAct | 10/10 | Полный разбор — лучший результат |

## Победитель: ReAct (10/10)
Шаги: понять проблему → оценить серьёзность → выбрать тон → написать ответ.
Результат: модель не упустила ни одной детали.

## Ключевые выводы

1. Baseline даёт 6/10 на простой известной задаче — модель справляется без промпта.
2. Role + Specificity без примера дали хуже чем baseline — промпт срезал длину но не исправил логику.
3. Few-shot исправил конкретную ошибку (контакты) лучше чем просто Role.
4. CoT добавил человечности — анализ причин виден в ответе.
5. ReAct дал максимум — пошаговый разбор не позволил пропустить детали.

## Главный урок
Чем сложнее задача с несколькими требованиями — тем важнее структурированная техника.
Для простых задач baseline работает. Для задач где важна каждая деталь — ReAct.
