# RAG Prompt System — База знаний интернет-магазина

Система промптов для RAG (Retrieval-Augmented Generation).
Модель отвечает строго по базе знаний и не придумывает информацию.

## Что такое RAG

Обычная модель отвечает из своих общих знаний.
RAG система передаёт модели релевантные документы вместе с вопросом.
Модель отвечает только по этим документам.

Схема:
Вопрос → поиск в базе → передача найденного в промпт → ответ только по базе

## Роль Prompt Engineer в RAG

- Проектирует промпт который заставляет модель отвечать только по базе
- Задаёт поведение при отсутствии информации
- Предотвращает галлюцинации через явные ограничения

## Финальный промпт

Ты ассистент поддержки интернет-магазина одежды.
Отвечай ТОЛЬКО на основе предоставленной базы знаний.
Если ответа в базе знаний нет — скажи: "Эта информация недоступна.
Обратитесь к менеджеру."
Не придумывай информацию которой нет в базе.

База знаний:
---
[вставляется релевантный фрагмент из базы]
---

Вопрос клиента: [вопрос]

## Тесты

### Тест 1 — Вопрос есть в базе

База знаний содержит:
Стоимость доставки: бесплатно при заказе от 5000₽, иначе 299₽.

Вопрос: Сколько стоит доставка если заказываю на 3000 рублей?

Ответ модели:
Если сумма заказа меньше 5000₽, доставка стоит 299₽.

Результат: ✅ точный ответ по базе, без лишнего

### Тест 2 — Вопрос не в базе

Вопрос: Есть ли у вас подарочная упаковка?

Ответ модели:
Эта информация недоступна. Обратитесь к менеджеру.

Результат: ✅ модель не придумала ответ, честно направила к менеджеру

## Ключевые правила RAG промпта

**1. Явное ограничение источника.**
"Отвечай ТОЛЬКО на основе предоставленной базы знаний."
Без этого модель смешивает базу со своими общими знаниями.

**2. Чёткое поведение при отсутствии данных.**
"Если ответа нет — скажи что информация недоступна."
Без этого модель галлюцинирует вместо признания незнания.

**3. Явный запрет на придумывание.**
"Не придумывай информацию которой нет в базе."
Дополнительный барьер против галлюцинаций.

## Результаты

| Галлюцинации | ✅ отсутствуют |

## Применение

- Чат-боты поддержки на основе FAQ
- Внутренние ассистенты по документации компании
- Голосовые роботы с ограниченной базой знаний
- Любая система где модель должна отвечать только по своим данным
